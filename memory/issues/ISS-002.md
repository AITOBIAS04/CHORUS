## Summary

token-report (daily 06:00 UTC) and fetch-tweets (daily 06:30 UTC) have failed to run on Jun 21, 22, and 23 — 3 consecutive days. cron-state.json shows `consecutive_failures=0` for both, meaning the scheduler is not dispatching them at all (no attempt = no failure recorded).

## Root cause hypothesis

messages.yml (the skill scheduler) is not dispatching morning-window skills. Possible causes:
- messages.yml cron is not firing in the 05:00–07:30 UTC window
- A workflow or runner quota issue blocking morning dispatches
- The scheduler condition logic has a bug affecting early-morning skills

## Remediation

1. Check messages.yml workflow run history for the Jun 21–23 morning window
2. Verify cron schedule in messages.yml for token-report and fetch-tweets
3. Manual re-run of token-report and fetch-tweets if dispatch is available
4. For weekly-shiplog: investigate why Monday 09:00 cron is not firing
